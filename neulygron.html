Key finding

Non-financial PDFs trigger classification

Bug #1 · Non-financial PDF classified without validation

The app accepted any PDF upload (for example, a book or any non-financial document) and, when clicking Classify, still triggered the AI request and returned classification results. The output was random and incorrect, presented as if it were a valid expense breakdown.

Probability: 5 / 5 Impact: 5 / 5 Risk score: 25 Very high

Why this matters: Neulygron works with people's personal finances. Wrong classifications can mislead users about their spending and affect their personal economy. The probability is high because PDF upload and classification is the core flow, and uploading the wrong file by mistake is easy.

See marker #1 in the risk matrix (top-right extreme cell).

Proposed measures

Mitigations I recommended to the developer

To reduce the chance of this issue and limit its impact, I proposed two complementary safeguards:

  • 1. Limit PDF upload size

    Reject or block files above a reasonable size threshold before they reach the AI pipeline. This prevents oversized documents from being processed and reduces token consumption costs for the company.

  • 2. Add a safeguard in the AI prompt

    Update the classification prompt so the model checks whether the uploaded document is actually a financial report. If it detects a non-financial PDF, it should refuse to classify and return a clear message to the user instead of producing random results.

Together, these measures mitigate the risk of wrong classifications, protect users' financial data integrity, and lower AI processing costs by avoiding unnecessary requests on invalid or oversized files.
